# Contributing

Thanks for your interest in improving this project.

## Workflow

1. Fork the repository  
2. Create a feature branch: `git checkout -b feature/short-name`  
3. Make focused commits with clear messages  
4. Ensure the app still runs locally  
5. Open a Pull Request describing **what** and **why**

## Guidelines

- Keep PRs small and reviewable  
- Do not commit secrets, `.env`, `venv/`, or OS junk (`.DS_Store`)  
- Match existing code style  
- Update the README when behavior or setup changes  

## Reporting issues

Include:

- Steps to reproduce  
- Expected vs actual behavior  
- OS / Python / Node versions  

## Code of conduct

Be respectful and constructive. Harassment or spam will not be tolerated.
